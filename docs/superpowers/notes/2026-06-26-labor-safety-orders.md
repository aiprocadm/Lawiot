# Архив приказов по охране труда (2026-06-26)

Пользователь передал 66 ссылок publication.pravo.gov.ru и попросил завести их в
корпус («архив + текущие»). Все 66 оказались приказами Минтруда «Об утверждении
Правил по охране труда…» 2014–2020 гг.

## Ключевые находки (durable)

1. **Все 66 на 2026 г. УТРАТИЛИ СИЛУ.** Батч 2020 г. («регуляторная гильотина»)
   содержит оговорку «вступает в силу с 1 января 2021 года и **действует до
   31 декабря 2025 года**» → истёк. Приказы 2014–2018 гг. отменены этим же
   батчем 2020 г. (напр., 781н-2020 «Признать утратившим силу… 722н-2015»).
   Поэтому status=**repealed** для всех (слой правового статуса покажет
   «утратил силу» — иначе нарушили бы честность статуса, см. #70).
2. **PDF на publication.pravo.gov.ru — скан без текстового слоя** (pypdf даёт
   0 символов на 12- и 49-страничных файлах). OCR не делаем (тяжёлая
   зависимость против low-maintenance). Подтверждает прежнее «PDF там — сканы».
3. **Но в ИПС есть консолидированный HTML** для большинства: `doc_itself&print=1`
   отдаёт текст, а существующий `parse_points` (doc_type="order", #41) разбирает
   его на пункты/приложения (300–1452 пункта/акт, «сирот» 2–3 — это собственные
   пункты 1./2./3. приказа до «Приложения», норма).

## Метод дискавери nd= (durable)

ИПС-поиск по-прежнему 500 headless, но **через Claude-in-Chrome работает**:
1. Браузерный intelsearch `?searchres=&bpas=cd00000&intelsearch=<win-1251,+вместо%20>`
   (ВАЖНО: пробелы — `+`, не `%20`, иначе «Неверные параметры запроса»).
2. Список результатов — во вложенном фрейме; ссылки результатов несут nd в `href`
   (digits). Выставив «выводить по: 100» и сматчив по (номер+дата) из заголовка
   результата против publication-метаданных (api/Documents), получили nd оптом.
3. Каждый nd проверен живым headless `fetch+parse(order)`: заголовок «…охране
   труда…», пункты без аномалий. 58 из 66 подтверждены.

## Итог

- **SAFETY_ORDER_ACTS = 58** приказов (slug `ot-<номер>-<год>`), status=repealed,
  doc_type=order, источник ИПС doc_itself&print=1, auto_ingest+auto_publish=True
  (sweep один раз заингестит/опубликует статичный текст; change-detection далее
  замораживает). Заводятся командой `seed_corpus` (агрегирует labor_law +
  labor_safety_orders).
- **SAFETY_PENDING_ACTS = 8** приказов (1104н/1101н/1100н-2014, 336н/439н/552н/
  551н-2015, 922н-2020): в ИПС консолидированного HTML нет (только скан-PDF) →
  PendingAct со ссылкой на publication. Завести в SEED, когда найдётся nd= ИПС.

## Расширенный список (143 → 111 уникальных, 2026-06-26)

Пользователь дополнил до 143 ссылок. Характеризация через publication `api/Documents`
показала разнородность: 58 уже в SAFETY_ORDER_ACTS; ~16 — **поправки** (наша
авто-консолидация уже вшивает их в сводный текст); ~18 — **региональные конкурсы**
(детский рисунок / «Лучший специалист по ОТ» / смотры — не нормативные); ~5 — оффтоп
(296-ФЗ парниковые газы, 264 учёт товаров и т.п.); 426-ФЗ уже в корпусе. Решение
пользователя: «нормативные + фед. поправки» (без регионалок и оффтопа).

Отфильтрованный ценный остаток (7) добавлен в `SAFETY_PENDING_ACTS` (nd= ИПС ещё не
разрешён — Claude-in-Chrome в этой сессии был нестабилен/зависал):
