# Spike: pymorphy-лемматизация поиска (§17) — результаты и рекомендация

Дата: 2026-06-11. Вопрос из §17 спеки: даст ли «более точная лемматизация
поиска (pymorphy)» реальный прирост над текущим Postgres FTS
(`config="russian"`, snowball-стеммер)? Проверено эмпирически на живом
корпусе: опубликованный ТК РФ, 424 статьи (slug `tk-rf`); ground truth —
word-boundary regex по тексту статей. БД не изменялась (только чтение).

## Текущий поиск (кратко)

`search/services.py:search_documents`: запрос → `SearchQuery(config="russian",
search_type="websearch")` против предвычисленных `search_vector`
(Redaction: title A + full_text B; Article: number+title A + text B; GIN,
миграция `0005`, пересборка `reindex_search`). Топ-100 на источник по
`SearchRank`, схлопывание до лучшего совпадения на документ, сниппет —
`ts_headline` с сентинел-подсветкой.

Snowball — алгоритмическое отрезание суффиксов без словаря: ё→е сводит сам
(«отчёт»/«отчет» — один лексем), регулярные падежи/числа ок, но не знает
супплетивных парадигм, беглых гласных и основ на -и.

## Измеренные промахи (12 кейсов, 139 промахов на уровне статей)

| Класс | Примеры | Промахов | Закроет pymorphy |
|---|---|---|---|
| Словоизменение (супплетивы, беглые гласные, и-основы) | «ребенок» → 0/13 (стем `ребенок` ≠ `ребенк` косвенных форм); «мать» → 0/4 (стем `мат`); «дети» 17/24; «люди» 5/13; «забастовка» 15/16 (мимо «забастовок»); «простой» мимо ст. 157/414/74 | 44 | **44/44 (100%)** |
| Словообразование | «уволен»/«уволить» → 2/42 (стем `увол` ≠ `увольнен`); «отчетность» | 42 | ~4 (10%) |
| Синонимы/аббревиатуры | «зарплата» → 0/~49 (в ТК — «заработная плата»); «МРОТ» → 0/2; «декретный отпуск» → 0/6 | 57 | 0 |

Итого query-time расширение pymorphy3 закрывает ~48/139 (~35%), причём
именно тот класс, где сейчас **ноль** результатов на естественные запросы.

Эмпирика pymorphy3 (2.0.6 + dicts-ru, **уже стоит в venv**, в коде не
используется): init `MorphAnalyzer` 0.72 с на процесс; расширение слова
1–9 мс; лексема 8–51 форма; OR-tsquery из ~50 форм — 14–116 мс против 8 мс
у базового (на 424 статьях незаметно).

## Рекомендация: внедрять частично

Query-time расширение + мини-словарь синонимов, всё в `search/services.py`,
без переиндексации и без кастомных словарей Postgres:

1. Ленивый синглтон `MorphAnalyzer()`.
2. На слово запроса: разборы со `score ≥ 0.03` среди топ-6 (порог важен —
   именной разбор «простой» имеет score 0.065 и не входит в топ-3), собрать
   словоформы лексем, фильтр `^[а-яё-]+$`, ё→е.
3. `SearchQuery(..., search_type="raw")` вида `(ф1|ф2|…) & (…)`, объединить
   через `|` с текущим websearch-запросом — кавычки/операторы websearch
   сохраняются, расширение только добавляет recall; ranking остаётся
   `ts_rank` (это же смягчает шум от омонимии).
4. Туда же — статический словарь синонимов (~10 строк): зарплата → заработная
   плата; декрет(ный) → отпуск по беременности и родам; МРОТ → минимальный
   размер оплаты труда; увольнение ⇄ уволить. Закрывает класс, который
   морфология не закрывает в принципе (60% промахов).

Зависимости/риски: pymorphy3 уже в venv (добавить в requirements); чистый
Python + словари OpenCorpora ~15 МБ диск / ~30–50 МБ RSS; Windows ок
(проверено в этом окружении); сопровождение ~0.

Почему НЕ hunspell/ispell в Postgres: лечит тот же класс на этапе
индексации, но требует файлы в `$SHAREDIR/tsearch_data` контейнера
(кастомный образ/volume — ломается при апгрейде образа), кастомную text
search config и перестройку индексов в каждом окружении — ровно та
«обслуживаемая инфраструктура», которой проект избегает.

## Апдейт 2026-06-13: расширение словаря синонимов

Внедрено (PR #20) + расширено: к исходным 6 ключам добавлены разговорные
термины, чей стем НЕ совпадает с юридической формулировкой (иначе snowball
уже находит её): `больничный`→«временной нетрудоспособности», `отгул`→«день
отдыха», `удаленка`/`удаленный`→«дистанционная работа», `вахта`→«вахтовый
метод», `контракт`→«трудовой договор». Каждая фраза проверена на присутствие
в живой фикстуре `tk_rf_real.html` word-boundary-regex'ом (нетрудоспособность
19, день отдыха 10, дистанционная 100, вахтовый метод 29, трудовой договор
746). Сознательно НЕ добавлены `командировка`/`сверхурочные`/`ненормированный`:
их стем совпадает с легальной формой, базовый запрос их и так находит.
