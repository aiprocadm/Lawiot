## ТК РФ — `ingestion/fixtures_raw/tk_rf_real.html`

**Рабочий URL (полный консолидированный текст):**
```
http://pravo.gov.ru/proxy/ips/?doc_itself=&nd=102074279&page=1&rdk=0
```
- `nd=102074279` — идентификатор ТК РФ в ИПС.
- **Ключевой нюанс источника:** базовый URL `?docbody=&nd=102074279` отдаёт НЕ текст, а
  *селектор редакций* (список ~140 актов-поправок «1 - от 24.07.2002 № 97-ФЗ (изм.)…»).
  Полное тело документа отдаёт только эндпоинт `?doc_itself=&nd=...&page=1&rdk=0`.
- **Кодировка: `windows-1251`** (не UTF-8). `html_to_text` через BeautifulSoup(`html.parser`)
  определяет её из meta-тега и декодирует корректно — проверено.
- Размер: 492 189 байт.

**Наблюдаемая структура (через `html_to_text` + регэкспы):**
| Элемент | Кол-во | Формат строки (реальный) |
|---|---|---|
| Разделы | **14** | `РАЗДЕЛ I. ОБЩИЕ ПОЛОЖЕНИЯ` … `РАЗДЕЛ XIV. ЗАКЛЮЧИТЕЛЬНЫЕ ПОЛОЖЕНИЯ` |
| Главы | **61** | `ГЛАВА 1. ОСНОВНЫЕ НАЧАЛА ТРУДОВОГО ЗАКОНОДАТЕЛЬСТВА` |
| Статьи | **424** | `Статья 1. Цели и задачи трудового законодательства` |

**Что сломалось в парсере (и почему):**
- ❌ **Разделы/главы — ВЕРХНИЙ РЕГИСТР** (`РАЗДЕЛ`/`ГЛАВА`), а `SECTION_RE`/`CHAPTER_RE`
  были регистрозависимы (`^Раздел`/`^Глава`) → 0 разделов, ~1 глава вместо 14/61.
  **Фикс:** регистронезависимое сопоставление разделов/глав (флаг `re.I`).
  Статьи — заголовочный регистр (`Статья N.`), `ARTICLE_RE` их ловит, трогать не нужно.
- ✅ **Заголовок** распознан верно: «Трудовой кодекс Российской Федерации» (`detect_title`
  поймал по ключевому слову «кодекс»).
- ⚠️ **`detected_number` = `10-ФЗ`** вместо `197-ФЗ`: наивный «первый матч `N-ФЗ`» поймал
  чужую ссылку в теле. Это лишь ПОДСКАЗКА куратору, `official_number` задаётся вручную —
  фикс не обязателен. (Возможное улучшение на будущее: искать номер рядом с заголовком/датой.)

**Приёмочные инварианты для теста Фазы 5** (консервативные нижние границы):
`sections >= 14`, `chapters >= 60`, `articles >= 400`, `title содержит "кодекс"`.

## Дополнение 2026-06-11: семантика `rdk`, обрезка ответа, надстрочная нумерация

**`rdk=0` — это НЕ актуальная редакция, а ИСХОДНЫЙ текст (как подписан).**
Захват выше (492 КБ, 424 статьи, без единой пометки «в редакции», без главы 49.1
о дистанционной работе) — редакция от 30.12.2001. Именно она была по ошибке
опубликована как «текущая». Правильные параметры `doc_itself`:

```
http://pravo.gov.ru/proxy/ips/?doc_itself=&nd=102074279&print=1
```

- **Без `rdk`** ИПС отдаёт *актуальную* редакцию (эквивалент максимального rdk;
  для ТК РФ на 2026-06-11 это `rdk=192`, правки до 31.07.2025). Явный `rdk` в
  seed-URL не указываем — иначе ссылка протухает с каждой новой поправкой.
- **`print=1` обязателен:** без него ответ обрезается на ~620 910 байтах
  (валидный HTML, но текст ТК РФ обрывается на ст. 181). Параметр `page`
  эндпоинт игнорирует (`page=1/2/all` — байт-в-байт одинаковые ответы).
- **Надстрочная нумерация:** дробные номера ИПС размечает как
  `Статья 312<span class="W9">1</span>.` (= ст. 312.1). `get_text("\n")`
  рвал такой заголовок на три строки → парсер вообще не видел дробные
  статьи/главы. Фикс в `html_to_text`: замена `span.W9` на `.N` + `soup.smooth()`.
- Перезахваченный `tk_rf_real.html` (1 459 870 байт): **15 разделов, 70 глав,
  538 статей (114 дробных), 0 сирот**; `detected_number` теперь ловит «97-ФЗ»
  из ранней пометки «в редакции…» — по-прежнему лишь подсказка куратору.

## PDF-акт

В этой сессии PDF-акт трудового права с pravo.gov.ru не захватывался (ТК РФ доступен как
HTML по эндпоинту выше). Фаза 3 (PDF) реализуется, когда в корпус добавится акт, который
источник отдаёт только как PDF; тогда захватить его `capture_fixture` и заполнить
`test_content_to_text_extracts_pdf` известной фразой. Официальный PDF ТК РФ как запасной
источник существует (напр. `pd.rkn.gov.ru/docs/Trudovoj_kodeks_RF.pdf`).
