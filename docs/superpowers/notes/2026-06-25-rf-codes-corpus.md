# Загрузка всех кодексов РФ в корпус (2026-06-25)

Расширение корпуса с 4 актов до 4 + **23 действующих федеральных кодекса РФ**.
ТК РФ уже был в корпусе (отдельной записью с `auto_publish`).

## Как найдены источники (durable)

ИПС-поиск pravo.gov.ru по-прежнему недоступен headless (фреймсет → 500), поэтому
`nd=` нельзя резолвить по названию. Рабочий путь дискавери:

1. **Официальный индекс кодексов** `http://pravo.gov.ru/codex/` отдаёт полный
   авторитетный список всех 23 кодексов с ФЗ-номером и датой подписания (UTF-8).
   Сами ссылки ведут на `actual.pravo.gov.ru` (JS-hash-вьюер, для нас бесполезен),
   но это эталонный СПИСОК + даты подписания → в `sign_date` сида.
2. **`nd=` для ИПС** найдены `WebSearch` с `allowed_domains=["pravo.gov.ru"]`
   (выдаёт `proxy/ips/?docbody=&nd=...` для каждого кодекса). Коммерческие СПС
   как источник ТЕКСТА запрещены (§2), но целочисленный `nd=` — это факт, не текст.
3. **Верификация (обязательный гейт):** каждый `nd=` прогнан через живой
   `fetch(doc_itself&nd=X&print=1)` + `parse_document(doc_type="code")`. Принимался
   только если детектнутый заголовок совпал с ожидаемым кодексом И иерархия
   раздел/глава/статья без «сирот». Сеять непроверенный `nd=` запрещено (риск порчи
   корпуса) — все 23 проверены.

## Решения

- **`doc_type="code"`** для всех; **`level="federal"`**, **`source_status="official"`**.
- **`auto_ingest=True`, `auto_publish=True`** — режим авто-консолидации, как у ТК РФ.
  Ежедневный sweep сам публикует свежую сводную редакцию под гейтом
  `AUTOPUBLISH_MIN_RATIO=0.8` (защита от обрезанного ответа). Founder выбрал полностью
  автоматическую консолидацию (spec 2026-06-13). При желании флаг легко снять по акту.
- **`sign_date`** — из официального индекса; **`official_pub_date` НЕ заполняем**
  (нет надёжного источника на все 23; не выдумываем реквизит-идентичность).
- Замечание: `official_number` не уникален — ГК ч.3 и НК ч.1 оба «146-ФЗ» (разные
  годы). Ключ — `slug`; на отображение/линковку не влияет (различаются по title/slug).
