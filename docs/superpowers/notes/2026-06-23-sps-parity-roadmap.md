# Roadmap: путь к полноценной НСС-альтернативе (Гарант / Консультант / Контур.Законодательство)

Дата: 2026-06-23. Живой документ — обновлять по мере закрытия треков.

## 1. Стратегия (зафиксирована ранее, брейнштормы 06-14 / 06-17)

НЕ догонять классические СПС по **объёму** (миллионы документов, штат редакторов — нереально
для нашего масштаба), а быть **лучшим в нише трудового права** через **глубину + AI**.
Дифференциатор — AI-ассистент с заземлением в корпус (классические СПС только подступаются к
этому). Полнота корпуса — догоняем точечно по трудовому праву, не «всё подряд».

«Полноценная альтернатива» в нашей трактовке = в нише трудового права пользователь получает:
актуальный текст актов + поиск + связи + изменения + **обоснованный AI-ответ** не хуже, чем в
большой СПС, плюс уникальную глубину (внутренние ссылки, diff редакций, консолидация).

## 3. Треки

### A. Объём корпуса трудового права — НАИБОЛЬШИЙ прирост полноты
- **Блокер: источник.** ИПС pravo.gov.ru (`doc_itself&nd=<ID>&print=1`) даёт чистый HTML, но
  поиск `nd=` headless недоступен (фреймсет→500; подтверждено httpx + Claude-in-Chrome).
  `publication.pravo.gov.ru` имеет JSON-API (PR #42), но PDF там — сканы без текста.
- **Что нужно от user'а:** на каждый целевой акт — `nd=`-URL из реального браузера, ЛИБО
  подключить Claude-in-Chrome (тогда обнаружение можно автоматизировать).
- **Инфраструктура готова:** `seed_corpus`, `ingest_url`, авто-sweep, авто-консолидация,
  semi-auto discovery (PendingAct + bind_to_ips). Дать ID — корпус наполняется конвейером.
- Кандидаты ниши: 426-ФЗ (есть), 10-ФЗ/82-ФЗ (есть), + постановления/приказы Минтруда, ФЗ о
  занятости, ключевые подзаконные акты по охране труда, оплате, отпускам.

### B. Судебная практика — ключевой раздел классических СПС
- **Блокер: источник.** Нет открытого API (sudact.ru, ГАС «Правосудие» — антибот/без API;
  КонсультантПлюс/Гарант запрещены §2). Требует решения user'а по легальному источнику.
- При наличии источника: новая модель `CourtDecision` + парсер + связь «практика к статье».

### C. AI-глубина (срезы поверх PR #47 — нужен merge #47)
- Срез 2: **стриминг ответа** в UI (SSE/htmx) — мгновенная отзывчивость.
- Срез 3: **многоходовой диалог** (история сессии, уточняющие вопросы).
- Срез 4: **семантический поиск** (pgvector + эмбеддинги) — ловит перефразировки, которых не
  берёт FTS; гибрид с текущим лексическим поиском.
- Срез 5: **AI-объяснение diff редакций** («что изменилось и как это влияет»).
- Все стекаются на #47 → сначала merge.

### D. Reader/навигация-глубина (бóльшей частью РАЗБЛОКИРОВАНО)
- **Межактовые гиперссылки** «N-ФЗ» / «Трудовым кодексом» в тексте → страница акта в корпусе
  (расширение #49 на cross-doc; переиспользует `ingestion/links.py:find_citations` +
  `CODEX_PATTERNS`). NB: трогает ту же строку `_article_node.html`, что #49 → делать ПОСЛЕ
  merge #49 (иначе конфликт).
- **Версия для печати** акта/статьи (`@media print` + чистый шаблон) — независимо.
- **Редакция на дату** — нужна 2-я опубликованная редакция (демон поймает реальную поправку).
- Закладки/заметки пользователя — нужна модель+миграция (личный кабинет).

### E. Редакторский слой (нужны люди)
- Комментарии/аннотации к статьям, обзоры — контент, который пишут юристы-редакторы. Вне
  автономного scope; продуктовое решение user'а (нанимать/писать самому).

### F. Надёжность/мониторинг (в осн. готово)
- Авто-консолидация (#26/#27) + B1 hardening (нормализованный хэш) на main. Демон qcluster
  `docker compose up -d` для боевого daily-sweep — инфра-хвост.

## 4. Граф зависимостей (что разблокирует что)

```
merge #46–#49 ──► AI-срезы 2-5 (C) ; межактовые ссылки (D) после #49
nd=-источники / Claude-in-Chrome ──► объём корпуса (A) ──► «полнота» ниши
                                         └► 2-я редакция со временем ──► редакция-на-дату
легальный источник судебки ──► трек B (court practice)
юристы-редакторы ──► трек E (комментарии)
```

## 5. Что нужно от пользователя (явные действия)

1. **Смержить PR #46–#49** (Claude не мержит в main) — разблокирует трек C и межактовые ссылки.
2. **Дать `nd=`-URL'ы** актов трудового права (из браузера) ИЛИ подключить **Claude-in-Chrome**
   — разблокирует трек A (объём), самый большой прирост к «полноценности».
3. **Решить по источнику судебной практики** (легальный) — разблокирует трек B.
4. (Опц.) Решить по редакторскому контенту (трек E).
