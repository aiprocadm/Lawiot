# Сравнение редакций для читателя (reader diff) — Design

**Дата:** 2026-06-10. **Статус:** утверждено пользователем (вариант A — постатейный inline-diff).
**Контекст:** первая фича пост-v1 (§17 основной спеки «Сравнение редакций для читателя (diff)»).
Корпус: ТК РФ опубликован (редакция #3, `is_current`); вторая редакция появится от ежедневного
auto-sweep после реального изменения источника — фича строится forward-looking и проверяется
тестами на синтетике.

## 1. Что делаем

Читатель на странице акта видит список прошлых опубликованных редакций. Каждая из них —
ссылка на страницу «что изменилось: эта редакция → текущая». Diff постатейный, в git-стиле
(`+` строка появилась в текущей, `−` строка убрана), показываются только изменённые,
добавленные и удалённые статьи; неизменённые (`same`) скрыты.

Переиспользуется без изменений чистая функция `documents/diffing.py:diff_articles`
(уже используется в admin-diff куратора, План 3d).

## 2. Компоненты

| Компонент | Файл | Действие |
|---|---|---|
| Маршрут `GET /doc/<slug>/diff/<int:from_pk>/`, name=`redaction_diff` | `config/urls.py` | Add |
| View `redaction_diff` (`@login_required`) | `documents/views.py` | Add |
| Шаблон страницы diff (extends `base.html`, Pico) | `templates/documents/redaction_diff.html` | Create |
| Точка входа: «Другие редакции (N)» → ссылки на diff | `templates/documents/document_detail.html` | Modify |
| View-тесты | `documents/tests/test_views.py` | Add |

## 3. Поведение view

1. `document = get_object_or_404(Document, slug=slug)`.
2. `current` = опубликованная `is_current` редакция документа; нет → 404 (как в `document_detail`).
3. `older = get_object_or_404(... pk=from_pk, document=document, review_status=PUBLISHED)`;
   чужая/неопубликованная/несуществующая → 404. `older.pk == current.pk` → 404
   (сравнивать редакцию с собой бессмысленно; в UI такая ссылка не отображается).
4. `diffs = diff_articles(older.articles.all(), current.articles.all())` — направление
   старая → текущая: `+` = появилось в текущей, `−` = убрано из старой.
5. В шаблон передаются только `d.status != "same"`; если изменений нет — страница
   с сообщением «Текстовых изменений между этими редакциями нет».

Замечание о направлении: admin-diff сравнивает «текущая → черновик» (что войдёт при
публикации), читательский — «старая → текущая» (что уже изменилось). Оба — корректные
применения одной функции `diff_articles(базовая, новая)`.

## 4. Шаблон

- Шапка: «Изменения: редакция от {older.redaction_date} → текущая от {current.redaction_date}»,
  ссылка назад на акт.
- Блок на статью: заголовок «Статья {number} — изменена/добавлена/удалена», затем `<pre>`
  со строками diff; CSS-подсветка `+`/`−` — тот же минимальный стиль, что в admin-diff
  (зелёный/красный фон строки).
- В `document_detail.html` в `<details>` «Другие редакции»: непосредственно у каждой
  не-текущей редакции ссылка `{% url 'redaction_diff' document.slug r.pk %}`
  «что изменилось к текущей»; текущая помечена как сейчас.

## 5. Данные, безопасность, ошибки

- Чтение-только, ноль записей в БД, без новых зависимостей и моделей.
- `@login_required` — как весь просмотрщик (§10: внутренний справочник).
- Видны только опубликованные редакции (черновики недоступны читателю даже подбором pk → 404).
- Текст экранируется шаблоном Django по умолчанию (autoescape), как в admin-diff.

## 6. Тестирование (TDD, синтетика, фабрики)

View-тесты в `documents/tests/test_views.py` (паттерн существующих):
1. Изменённая статья → блок «изменена», есть `+`-строка нового текста и `−`-строка старого.
2. Добавленная статья → «добавлена»; удалённая → «удалена».
3. Неизменённая статья не отображается.
4. Аноним → redirect на login.
5. 404: черновик в `from_pk`; редакция чужого документа; `from_pk == current.pk`.
6. Точка входа: на странице акта с 2 опубликованными редакциями есть ссылка на diff;
   с 1 редакцией — ссылок нет.

## 7. Сознательно НЕ в v1 (YAGNI)

- Произвольное сравнение «любая ↔ любая» (два дропдауна) — только «старая → текущая».
- Пословная (intra-line) подсветка — только построчная.
- Diff заголовков/реквизитов редакций — только текст статей.
- Учёт переименования/перенумерации статей — выравнивание строго по `number`
  (ограничение унаследовано от `diff_articles`).
