# Дизайн: читаемые панели связей на карточке акта

Дата: 2026-06-16
Статус: одобрен (доводка v1, без новой scope)

## Проблема

Страница поиска (`templates/search/_results.html`) показывает читаемое
название акта (`r.document.title`), а панели связей в `<aside>` карточки
акта (`templates/documents/document_detail.html`) показывают **голый
`official_number`** («426-ФЗ»). Несогласованность + три конкретные
шероховатости:

1. **Загадочные подписи.** Ссылка на акт корпуса выводится как «426-ФЗ»
   вместо «О специальной оценке условий труда (426-ФЗ)».
2. **Скрытый баг пустой ссылки.** `{{ link.to_document.official_number }}`
   это весь текст `<a>`. `Document.official_number` объявлен `blank=True`,
   поэтому акт без номера рендерит **пустую, невидимую кликабельную
   ссылку**.
3. **Дублирующий префикс.** Каждый `<li>` под `<h3>`«Ссылается на»
   повторяет «Ссылается на:».
4. **Нет пустого состояния.** Читатель без подтверждённых связей видит три
   одиноких заголовка над пустыми `<ul>` (лента изменений в аналогичном
   месте пишет «Изменений пока нет.»).

## Решение

Минимальное, шаблон + одно свойство модели. **Без полей модели, без
миграции.**

### 1. `Document.reference_label` (свойство, `documents/models.py`)

Человекочитаемая подпись акта для панелей связей:

- `official_number` задан → `"{title} ({official_number})"`;
- `official_number` пуст → `"{title}"`.

Свойство (не метод), без обращений к БД, чистое. Закрывает баг №2 в корне:
текст ссылки больше никогда не пуст, пока есть `title` (а `title` —
`TextField` без `blank=True`, всегда заполнен у акта корпуса).

### 2. Применить во всех трёх панелях (`document_detail.html`)

- «Изменяющие / изменённые акты» (`amendments`): для `link.to_document` —
  `reference_label`; иначе `raw_citation` как сейчас.
- «Ссылается на» (`references`): аналогично; для цели вне корпуса —
  `raw_citation` + «(вне корпуса)» как сейчас.
- «На него ссылаются» (`incoming`): `link.from_document.reference_label`.

Длинные названия кодексов укорачиваем `|truncatechars` в шаблоне (порог
~80), чтобы панель не растягивалась.

### 3. Убрать дублирующий префикс

В `<li>` панели «Ссылается на» убрать текст «Ссылается на:» — заголовок
`<h3>` уже это сообщает. (В «Изменяющие» оставляем
`get_link_type_display`, т.к. он различает изменяет/изменён.)

### 4. Пустое состояние

Для каждой из трёх панелей: если список пуст — приглушённая строка-заглушка
«—» (`<li><small>—</small></li>` через `{% empty %}`), чтобы заголовок не
висел над пустотой.

## Что НЕ делаем (YAGNI / scope guard)

- Размещение связей на уровне статьи (`from_article`) — отдельная, более
  крупная фича, сознательно отложена.
- Изменения страницы поиска — она уже читаема; адаптация к
  `reference_label` возможна позже, вне этой задачи.
- Никаких полей модели и миграций.

## Тестирование

Расширить тесты карточки акта (`documents/tests/test_document_card.py` или
рядом, НЕ трогая hotspot `test_views.py`):

- `reference_label` с номером и без (чистый unit, без БД);
- панель «Ссылается на» рендерит `reference_label` цели из корпуса;
- цель с пустым `official_number` даёт **видимый** текст ссылки (название);
- пустая панель показывает заглушку «—»;
- внутри `<li>` «Ссылается на» нет дублирующего префикса.

Полный прогон `pytest` на Docker Postgres (`lawiot-db:5433`), `ruff check .`.

## Риск

Низкий. Шаблон + чистое свойство; обратимо; затрагивает только
read-only-просмотрщик за `@login_required`.
