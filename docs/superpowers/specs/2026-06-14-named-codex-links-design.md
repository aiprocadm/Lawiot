# Распознавание ссылок на кодексы по имени — дизайн

**Дата:** 2026-06-14
**Статус:** одобрено пользователем, ждёт плана реализации
**Файл реализации:** `ingestion/links.py` (+ тесты)

## Проблема

Авто-экстрактор связей (`ingestion/links.py:find_citations`) ловит только числовые
маркеры НПА вида `NNN-ФЗ`/`N-ФКЗ` (`CITATION_RE`). Российские законы в основном
тексте ссылаются на кодексы **по имени** («в соответствии с Трудовым кодексом»),
а не по номеру «197-ФЗ». Поэтому межактовые ссылки между «именованными» кодексами
системно не строятся.

Конкретный пробел (обнаружен 2026-06-13): 426-ФЗ «О специальной оценке условий
труда» **68 раз** упоминает «Трудовой кодекс», но ни разу — «197-ФЗ», поэтому
авто-связь 426-ФЗ → ТК РФ не возникает. Это граница эвристики §9, не баг ингеста.

## Решение (кратко)

Добавить в `ingestion/links.py` распознавание упоминаний кодексов по имени
(во всех падежах) и резолвить их в документы корпуса по `Document.title`.
Связь создаётся **только если кодекс уже есть в корпусе** (высокий сигнал, без
спама на ГК/НК/УК, которых в трудовом корпусе не будет). Без изменений схемы БД.

## Решения дизайна (зафиксированы с пользователем)

- **Резолвинг — только если кодекс в корпусе.** Если каноническое имя не
  резолвится в документ — связь НЕ создаётся (в отличие от `NNN-ФЗ`, где для
  нерезолвленных номеров пишется `raw_citation`). Причина: нерезолвленные именные
  ссылки на внекорпусные кодексы (ГК/НК/УК/КоАП) завалили бы очередь куратора
  шумом, который никогда не станет кликабельным.
- **Охват — реестр основных кодексов РФ** (Трудовой, Гражданский, Налоговый,
  Уголовный, КоАП, Земельный, Жилищный, Семейный, Бюджетный и т.п.). При
  резолвинге «только-в-корпусе» лишние записи реестра безвредны: резолвятся лишь
  те, что реально в корпусе. Новый кодекс в корпусе подхватывается без правок,
  если его имя уже в реестре.

## Компоненты

### 1. `CODEX_PATTERNS` (константа модуля)
Список пар `(regex, canonical_title)`:
- `regex` — компилированный, `re.IGNORECASE`, терпимый к склонениям через стемы.
  Пример для ТК: `re.compile(r"\bтрудов\w+\s+кодекс\w*", re.I)` ловит «Трудовой
  кодекс», «Трудовым кодексом», «Трудового кодекса» и пр.
- `canonical_title` — строка-якорь для резолвинга по `Document.title`
  (напр. `"Трудовой кодекс"`).
- Каждый кодекс — отдельный точный паттерн (закрытый набор → низкая поддержка).
- КоАП — особый случай: «Кодекс … об административных правонарушениях» (слово
  «кодекс» спереди) — собственный паттерн и собственный способ резолвинга по title
  (`icontains="об административных правонарушениях"`), т.к. `istartswith` по
  «Кодекс» неустойчив.
- Паттерны с уточняющим словом (ГПК «гражданский процессуальный кодекс», УПК,
  АПК) не пересекаются с базовыми (ГК «гражданский кодекс»), т.к. базовый паттерн
  требует «кодекс» сразу после прилагательного.

### 2. `find_named_citations(text)` — чистая функция
Зеркало `find_citations`. Текст → список `NamedCitation(name, context)`:
- `name` — **каноническое** имя из реестра (не пойманная падежная форма).
- `context` — очищенный фрагмент ±`CONTEXT_WINDOW` (60) вокруг первого вхождения.
- Дедуп по каноническому имени (по одной `NamedCitation` на кодекс).
- Без БД/сети.

### 3. Резолвинг в `extract_links_for_redaction`
После существующего цикла по `NNN-ФЗ` добавляется цикл по именным цитатам:
- Для каждого `NamedCitation` найти документ корпуса по `canonical_title`
  (`Document.title__istartswith` для большинства; особый матчер для КоАП),
  исключая сам документ (`exclude(pk=...)`).
- Резолвится → создать связь `link_type=references`, `origin=auto`,
  `status=suggested`, `context=citation.context`.
- Не резолвится → ничего (только-в-корпусе).

## Ключевые гарантии

- **Дедуп с `NNN-ФЗ`.** Если акт цитирует и «197-ФЗ», и «Трудовым кодексом» — оба
  резолвятся в tk-rf, но существующая проверка `already`
  (`from_document`+`to_document`+`link_type`) не даст создать вторую связь. Ровно
  одна ссылка на цель.
- **Идемпотентность.** Именные связи тоже `auto+suggested` → попадают в
  существующий цикл сброса/пересоздания в начале `extract_links_for_redaction`.
  `confirmed`-связи куратора (напр. ручная ст.214 ТК РФ → 426-ФЗ) сохраняются.
- **Самоцитата.** ТК РФ в собственной преамбуле упоминает «Трудового кодекса» →
  self-exclude по `pk` (как уже сделано для номеров).

## Тестирование

- **Pure** (`find_named_citations`): падежные формы → одно каноническое имя; дедуп;
  игнор «трудовой договор»/«трудовая книжка» (нет слова «кодекс»); мульти-кодексный
  текст; корректный контекст.
- **django_db** (`extract_links_for_redaction`):
  - резолв в tk-rf, когда tk-rf в корпусе → одна `suggested`-связь;
  - **ноль связей**, когда кодекса в корпусе нет (только-в-корпусе);
  - дедуп: текст с «197-ФЗ» и «Трудовым кодексом» → ровно 1 связь к tk-rf;
  - no self-link: текст tk-rf с «Трудового кодекса» → 0 связей на себя.
- **Живая фикстура** `ingestion/fixtures_raw/sout_426fz_real.html` (68 упоминаний
  ТК): с tk-rf в корпусе → ровно одна резолвленная связь 426-ФЗ → ТК РФ.

## Сознательно вне scope (YAGNI)

- **Аббревиатуры** «ТК РФ»/«ГК РФ» — омонимичны и рискованны; текущий пробел
  закрывается полным именем. Будущее расширение.
- Связь на конкретную статью кодекса — оставляем document-level, как все авто-связи.
- Поле `aliases`/`short_name` на `Document` — не нужно, резолвим по `title`.

## Допущения

- Документы-кодексы titled каноническим именем в номинативе («Трудовой кодекс
  Российской Федерации»), как сделано в сиде для tk-rf — `istartswith` по
  каноническому имени это контракт резолвинга.
