# Слой правового статуса и честности (Р1) — дизайн

**Дата:** 2026-06-23
**Статус:** одобрено пользователем (брейншторминг)
**Связь с ТЗ:** реализует §0.1 Р1 (политика «нулевого расхождения» — честная пометка
происхождения текста), Р2 (поле уровня нормативки заложить сразу), КП-12 и НФТ-8
(дисклеймер + ссылка на первоисточник на всех материалах), §23 (поля модели
`source_status`, `level`, `text_status`).

## 1. Цель и границы

Привести Lawiot к центральному принципу полного ТЗ: **система честно помечает
происхождение текста, даёт ссылку на первоисточник и дисклеймер на всех материалах**,
а модель данных заранее знает об уровне нормативки (федеральный/региональный/
муниципальный).

Срез **полностью аддитивный**: добавляются поля (с дефолтами) и элементы UI.
Существующее поведение — парсинг, авто-консолидация, поиск, публикация — не меняется.

**Вне объёма** (отдельные крупные треки ТЗ, здесь не строятся):
- Движок консолидации §6 (битемпоральность, eId, operations, резолвер дат) — то есть
  реконструкция как *производитель* текста. Здесь добавляется только статус-поле и
  пометка; значение `reconstruction` зарезервировано для будущего/куратора.
- Региональное наполнение (поле `level` закладывается, но данные не заводятся).
- Байт-в-байт хеш-контроль отображаемого текста: парсер реструктурирует текст ИПС
  (дерево СЕ, якоря), поэтому отображение НЕ является побайтовой копией источника.
  Заявлять обратное было бы нечестно. Хеш `RawSource.text_hash` (B1) остаётся
  внутренним контролем неизменности *сырого* источника, не отображения.

## 2. Ключевые проектные решения

1. **`text_status` — на уровне `Redaction`, не `Article`.** Редакция целиком приходит
   из одного источника ИПС → весь её текст имеет одно происхождение. Пер-статейный
   статус (как `unit_version` в §23 ТЗ) избыточен для текущей архитектуры и противоречит
   low-maintenance. Решение зафиксировано.
2. **`text_status` ортогонален `review_status`.** `review_status` (draft/published) —
   редакционное состояние; `text_status` — происхождение текста. Поэтому
   `text_status` = только {`official`, `reconstruction`}; «черновик» уже живёт в
   `review_status`, два конкурирующих «draft» не плодим.
3. **Баннер минимальный** (выбор пользователя): бейдж «Официальная редакция» /
   «Автоматическая реконструкция» + ссылка на источник, без пояснений про
   структурирование. Основную честность несёт глобальный дисклеймер в футере.
4. **Дефолты несут бэкфилл.** Все существующие источники — pravo.gov.ru (официальные),
   все акты федеральные, весь текущий текст из официального сводного раздела ИПС →
   дефолты `official`/`federal`/`official` корректны для существующих строк, отдельная
   data-миграция не требуется.

## 3. Модель данных (одна миграция в приложении `documents`)

`Document`:
- `source_status` — `CharField(choices=[("official","Официальный источник"),
  ("unofficial","Неофициальный источник")], default="official", max_length=20)`.
- `level` — `CharField(choices=[("federal","Федеральный"),("regional","Региональный"),
  ("municipal","Муниципальный")], default="federal", max_length=20)`.
- `region_code` — `CharField(max_length=10, blank=True)` (код субъекта РФ; пусто на
  федеральном уровне).

`Redaction`:
- `text_status` — `CharField(choices=[("official","Официальная редакция"),
  ("reconstruction","Автоматическая реконструкция")], default="official",
  max_length=20)`.

Вложенные `TextChoices`-классы оформляются по образцу существующих
(`Document.DocType`, `Redaction.ReviewStatus`).

## 4. Установка значений (минимальная правка существующего кода)

- `ingestion/services.py:ingest_target` и `import_manual`: при создании черновика
  редакции проставлять `text_status=Redaction.TextStatus.OFFICIAL` (источник — официальный
  сводный раздел ИПС, Р1.1). Это единственное добавочное присваивание; логика пайплайна
  не меняется.
- `documents/seed/labor_law.py`: в `update_or_create(defaults=...)` явно указать
  `level="federal"`, `source_status="official"` для ясности (дефолты и так это дадут).
- `reconstruction` нигде автоматически не выставляется — резерв для куратора/§6.

## 5. UI (КП-12/НФТ-8 — «на всех материалах»)

- **`templates/base.html` — глобальный футер** (виден на всех страницах):
  > Lawiot — справочный навигатор по открытым правовым данным. Не является источником
  > официального опубликования и не заменяет юридическую консультацию.
  > Официальный источник: pravo.gov.ru.
- **`templates/documents/document_detail.html` (reader/карточка):**
  - бейдж происхождения текущей редакции по `text_status` («Официальная редакция» /
    «Автоматическая реконструкция») рядом с датой редакции;
  - ссылка «Официальный источник» → `document.source_url`, только если `source_url` задан;
  - строки паспорта: `Уровень` (`get_level_display`) и `Источник`
    (`get_source_status_display`).
- **`templates/documents/document_print.html`:** строка дисклеймера (печать — материал).
- **docx-экспорт (`documents/views.py:document_export_docx`):** абзац-дисклеймер в конце
  документа (ТЗ §15 «экспорт … с дисклеймером»).

Стиль бейджа происхождения — лёгкий, в `base.html` рядом с существующими `.status-badge`
(переиспользовать паттерн; новый класс, напр. `.text-status-reconstruction` для визуального
отличия реконструкции).

## 6. Тестирование (TDD)

Новый изолированный файл `documents/tests/test_legal_status.py` (hotspot
`documents/tests/test_views.py` не трогаем — он конфликт-горячий):
- футер-дисклеймер присутствует на списке актов, reader, поиске;
- бейдж происхождения рендерится согласно `text_status` (official → «Официальная
  редакция»; reconstruction → «Автоматическая реконструкция»);
- ссылка «Официальный источник» присутствует при заданном `source_url` и отсутствует
  при пустом;
- новые поля модели дефолтятся (`official`/`federal`/`official`);
- паспорт карточки показывает уровень и статус источника;
- (опц.) docx-экспорт содержит строку дисклеймера.

Прогон через `run-tests.ps1` (Docker Postgres) либо WSL-фолбэк, если Docker недоступен.

## 7. Риски и смягчение

| Риск | Смягчение |
|---|---|
| Leaf-конфликт миграций `documents` с параллельной сессией | Известный паттерн: после мержа `makemigrations --merge`; работаю в отдельном worktree |
| Переобещание официальности | Баннер минимальный без заявления байт-идентичности; футер явно «не источник офиц. опубликования» |
| Сломать существующий рендер reader | Все правки шаблонов аддитивны и под `{% if %}`; новые поля с дефолтами |

## 8. Критерии приёмки

- На всех страницах (список, reader, поиск, печать) виден дисклеймер (КП-12/НФТ-8).
- Карточка акта показывает уровень, статус источника, бейдж происхождения редакции и
  (при наличии) ссылку на официальный первоисточник.
- Новые поля присутствуют в модели и админке, существующие данные корректно
  забэкфилены дефолтами.
- Весь существующий набор тестов зелёный + новые тесты `test_legal_status.py`.
- `ruff check .` и `manage.py check` чисты.
